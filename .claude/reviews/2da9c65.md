- **`de9bfa3` Topic 04 DPP greedy MAP + MMR λ ramp** (spec §11 Topic 04 + Investigation 14 + Spec v1.2 SPEC-UPDATED):
  - Two orthogonal flag-gated additions; both default OFF, existing 104 tests unchanged.
  - **(a) MMR λ ramp** (per-swipe, hot path): 1-line addition `λ(t) = λ_base · min(1, |exposed|/N_ref)` in `compute_mmr_next`. λ computation hoisted outside candidate loop (micro-optimization). Encourages relevance-heavy at session start (sparse signal), diversity-heavy as exposure grows.
  - **(b) DPP greedy MAP** (`compute_dpp_topk(cards, like_vectors, k, q_override=None)`):
    - Wilhelm 2018 kernel form: `L = np.outer(q, q) * (alpha * S + (1.0 - alpha) * np.eye(nv))` ✓ matches docstring.
    - Chen 2018 Cholesky-incremental greedy MAP: maintains `d` (residual variance) and `c_mat` (Cholesky columns); per iteration picks `argmax(d)` among unselected, updates `d` via residual subtraction. Singularity (`d[best] < eps`) → terminate selection + pad with q-descending remaining.
    - Two-phase fallback: phase-1 (embedding/q fetch failure) → ids[:k]; phase-2 (Cholesky exception) → q-sorted top-k.
    - **Critical correctness fix**: embeddings fetched via `get_pool_embeddings(ids)` — NOT from card dicts (which don't include embedding field; would crash). Flagged by back-maker per commit body.
    - α default 1.0; clamped to `[0, 1]` (α>1 breaks PSD per Wilhelm).
    - q_override parameter: bypasses `[0.4, 0.95]` clip when caller provides RRF-rescaled values already in `[0.01, 1.0]` range (prevents corruption of fused signal at both tails).

- **`ebbafd2` Topic 02 ∩ 04 Option α composition** (Investigation 07 + Investigation 14 §q derivation):
  - Single integration point per Investigation 07: when BOTH flags ON, q_i in DPP's L-ensemble reads from RRF-fused (cosine_rank, rerank_rank) → min-max rescale to `[0.01, 1.0]` instead of pure cosine.
  - `views.SessionResultView` refactor:
    - Capture `candidate_ids_cosine_order` BEFORE any reorder.
    - Sentinel pattern: `rerank_rank_by_id = None` unless rerank produced a real reorder (new_order ≠ candidate_ids_cosine_order). Failure or no-op cases keep sentinel None → DPP falls back to cosine q.
    - Topic 04 block branches on sentinel: RRF rescale path (when set) OR cosine-q standalone path (when None).
  - RRF formula (Investigation 07): `K_RRF=60` hardcoded; `fused[bid] = 1/(60 + cosine_rank) + 1/(60 + rerank_rank)`.
  - Min-max rescale (Investigation 14): `q = 0.01 + 0.99 * (fused - fmin) / (fmax - fmin)`; `fmax==fmin` edge case → all q=0.5 (no div-by-zero).
  - Architectural deviation flagged in commit body: Investigation 07 pseudocode operates on raw cosine-60; this commit operates on post-MMR top-K (typically k=20) — the existing integration point for both Topic 02 and Topic 04 individually. Composition follows that established pattern. Bigger refactor (composition on raw cosine-60) flagged as out of scope.
