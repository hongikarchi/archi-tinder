## Security Analysis

In-depth walk of the auth / token / input-validation / authorization surface touched by this branch:

- **No new auth/authz surface**. Flag is server-side only (Django settings), not exposed via API.
- **No new request-body shape**. parse_query signature unchanged.
- **No new response shape**. parse_query return dict unchanged.
- **Telemetry sanitation**: `cache_name_hash` field stores the **8-char SHA-256 prefix** of the prompt, NOT the full Gemini resource name (e.g. `cachedContents/abc123`). PII-safe, doesn't leak internal Gemini URNs.
- **Django cache key**: `gemini_cache_name:archi-tinder-chat-{hash}` — server-internal, no auth surface. No collision with DRF throttling (different prefix).
- **Cache contents**: Gemini resource name string only. No PII, no tokens, no secrets.
- **Logger writes**: `logger.info` on cache create + `logger.warning` on failure — both sanitary (no secrets, no raw prompt content).
- **404 detection by string match** (`'404' in exc_str or 'NOT_FOUND' in exc_str`): not vulnerable to injection (we don't trust the string for any other purpose); a false-positive falls through to the uncached path → graceful degradation to a working response. Defense-in-depth.
- **MagicMock guard** (`isinstance(_raw_cached, (int, float))`): prevents non-serializable types from leaking into JSONField in tests with mocked Gemini responses. Good defensive coding.
- **No new external dependencies**: uses existing google-genai SDK 1.47.0; verified `CreateCachedContentConfig`, `GenerateContentConfig.cached_content`, `usage_metadata.cached_content_token_count` all exist.
- **Decision documented (proactive caches.get skip)**: spec criterion #2 specified `client.caches.get(name=cached_name)` validation after Django hit. Implementation skips this — adding ~50-100ms to EVERY parse_query is a bad latency trade vs. preventing a near-impossible event (Django expires first per 80% invariant). Reactive 404 path is the bounded fallback. Trade-off resolved correctly toward latency.

## Observations (sub-MINOR, not flagged as findings)

1. **`_get_prompt_hash()` recomputed every call**: `_get_cache_name()` and `_get_django_cache_key()` both call it. SHA-256 of a 5924-byte string is microsecond-level — no perf concern. Could be memoized as a module-global since `_CHAT_PHASE_SYSTEM_PROMPT` is import-time constant, but that introduces a stale-cache risk if the prompt is ever monkeypatched in a test. Current pattern is correctness-safer; not worth changing.

2. **404 detection by substring match**: `'404' in exc_str or 'NOT_FOUND' in exc_str` is brittle in theory — a different exception with the literal "404" in its message could trip the fallback. In practice the fallback degrades gracefully (uncached retry succeeds), so a false-positive is bounded. Future-proof would be SDK exception type detection (e.g., `isinstance(exc, NotFoundError)`) but the google-genai SDK doesn't expose this consistently. Acceptable as-is.

3. **Duplicated `_call` definitions in parse_query**: there are three `def _call`/`_call_uncached` definitions (cached path, uncached path, 404 fallback uncached path) with `# noqa: F811` on the elif branch. Minor duplication, but the flag-gated structure makes the branching explicit. Refactoring to a single helper with a `cached_content` parameter would slightly obfuscate the "either-or" structure. Acceptable as-is.

4. **80% TTL math edge case**: `int(ttl * 0.8)` — for ttl=1, Django TTL = 0 (cache.set with timeout=0 means "expire immediately" or "no timeout" depending on backend). Not reachable at production values (ttl=3600 → 2880; ttl=300 → 240; ttl=10 → 8). Documented for completeness.

5. **`@pytest.mark.django_db` Neon-lock note in commit body**: back-maker's reviewer initially flagged 9 django_db tests as failing due to Neon test DB lock during the test run. Spot-check confirmed they're @django_db not unit-level — environmental. I re-ran all 20 IMP-5 tests + the full 301-test suite in this review session, all pass cleanly. No issue.

These are notes for future awareness, not gaps that should block this push.
