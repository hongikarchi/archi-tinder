## Security Analysis

- **ParseQueryView input validation** is the headline security improvement: caps `conversation_history` length (10), text length (2000 chars), role allowlist {'user', 'model'} — prevents Gemini cost-amplification DoS and role-string-injection attempts. Validation runs **before** the Gemini call, so invalid input never reaches the upstream. Returns 400 with descriptive detail, not 500.
- **`emit_event` swallows exceptions** — explicit module docstring: "failure to log must not block the request". An attacker can't induce 500s by triggering emit failures (e.g., DB unavailable, payload too large). The function returns `None` and logs a warning.
- **No new endpoints exposed** — all logging is internal. SessionEvent is not serialized via DRF anywhere in this branch (it's read by future analytics, not by the client).
- **Migration 0009/0010**: SessionEvent FK to UserProfile uses `on_delete=SET_NULL` — preserves event records for analytics when a user is deleted (right call for audit/metric data).
- **`thinking_budget=0`** does not introduce a security surface — it's a model parameter, not user input. Cost reduction (no hidden reasoning tokens billed) is a side benefit.
- **Governance verification**: only `research/algorithm.md` touched by main pipeline (3 reporter narrow-exception syncs in this batch). No edits to `research/spec/`, `research/search/`, or `research/investigations/`. ✓

No vulnerabilities introduced.
